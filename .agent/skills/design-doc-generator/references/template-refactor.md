# 重构任务模板

适用于代码优化和重构。

## 模板内容

```markdown
# 你的任务

## 阶段一：现状分析

1. 分析【重点文件】中指定代码的问题：
   - 代码坏味道（重复代码、过长方法等）
   - 性能问题
   - 可维护性问题
2. 输出：问题清单和优先级排序

## 阶段二：重构方案（等待确认）

1. 针对每个问题提出重构方案
2. 说明：
   - 重构前后的对比
   - 风险评估
   - 是否需要分阶段实施
3. 等待我确认重构范围

## 阶段三：实施重构

1. 按确认的方案重构代码
2. 保证重构前后功能一致
3. 每个文件的改动点需说明

## 约束
- 不改变外部接口
- 保持向后兼容
- 每次重构聚焦一个点
```

## 使用说明

- 阶段二完成后必须等待用户确认重构范围
- 约束部分可根据用户需求调整，如用户允许修改接口则放开对应约束
- 重构应遵循小步快跑原则，避免一次性大规模改动
- 每次重构后需确保功能不变，可建议用户先补充测试用例
