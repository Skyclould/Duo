# Bug修复任务模板

适用于问题定位和修复。

## 模板内容

```markdown
# 你的任务

## 步骤1：问题定位
- 阅读【重点文件】，分析可能的问题原因
- 输出：问题分析报告，包含：
  - 问题复现条件
  - 可能的原因（列出2-3个）
  - 建议的排查方向

## 步骤2：确认根因（等待我确认排查方向）
- 按确认的方向深入分析
- 输出：根因确认，定位到具体代码行

## 步骤3：修复方案
- 提出修复方案
- 说明修复的风险和影响范围
- 等待确认

## 步骤4：实施修复（方案确认后执行）
- 修改代码
- 说明如何验证修复有效
```

## 使用说明

- 步骤1需要重点关注用户描述的问题现象和错误日志
- 步骤2等待用户确认后再继续
- 如果用户已经明确了根因，可跳过步骤1和2，直接从步骤3开始
- 修复完成后需要提供验证方法
