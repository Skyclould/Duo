# 代码Review/分析任务模板

适用于代码理解和学习。

## 模板内容

```markdown
# 你的任务

## 分析并讲解 [指定代码/方法]

请生成详细的分析文档，包含以下内容：

### 1. 概述
- 这段代码的核心功能是什么
- 在整体架构中的位置和作用

### 2. 逻辑流程
- 用流程图或步骤列表展示执行过程
- 关键分支和判断逻辑

### 3. 术语解释
- 代码中涉及的业务术语
- 变量/方法命名的含义

### 4. 设计要点
- 为什么这样设计
- 有哪些可以优化的地方

### 5. 使用示例
- 调用这段代码的典型场景
- 输入输出示例
```

## 使用说明

- `[指定代码/方法]`：替换为用户希望分析的具体代码、方法或类名
- 如果用户指定了多个代码段，为每个段分别生成分析
- 术语解释部分需要结合"我的情况"中用户的技术水平来决定讲解深度
- 可以根据需要增减子模块，例如添加"安全分析"或"性能分析"
