# 标准开发任务模板

适用于一般的功能开发需求。

## 模板内容

```markdown
# 你的任务

## 阶段一：分析与设计（必须等待确认再进入下一阶段）

1. 阅读【重点文件】中指定的代码，理解现有逻辑
2. 在 "[指定目录]" 生成设计文档，包含：
   - 需求理解：用自己的话复述需求要点
   - 现有逻辑分析：相关代码的当前实现方式
   - 改动方案：需要修改的文件和具体改动点
   - 影响范围：可能影响的其他模块
   - 测试计划：如何验证改动正确性

## 阶段二：编码实现（设计确认后执行）

1. 按照设计文档修改代码
2. 遵循以下约束：
   - 代码风格与现有代码保持一致
   - 仅修改设计文档中列出的文件
   - 添加必要的注释和日志
3. 修改完成后说明改动点

## 阶段三：验证与交付

1. 确保代码能够编译通过
2. 说明如何测试验证
```

## 使用说明

- `[指定目录]`：替换为用户指定的设计文档输出路径，如未指定可使用项目根目录下的 `docs/` 路径
- 阶段一完成后**必须等待用户确认**，才能进入阶段二
- 如果用户需求较简单，可以合并阶段二和三
